## 📌 What is BioSmart?

BioSmart is a **web-based AI prediction tool** that estimates biogas composition and yield from an anaerobic digester — the biological reactor that converts organic waste into clean energy. It takes two sets of inputs:

- **What goes in:** The type and percentage mix of waste materials (feedstock)
- **How it operates:** Digester conditions — temperature, digester Volumne, urea addition, retention time, and daily feed load

Given these inputs, BioSmart predicts digester run status, failure cause, total biogas yield, and methane content — all within seconds — and provides actionable improvement tips based on the predictions.

### Why Does This Matter?
Biogas from organic waste is one of the most promising forms of renewable energy, especially for agricultural and municipal waste management. Designing and operating a digester efficiently requires deep knowledge of microbiology, chemistry, and process engineering. BioSmart makes this knowledge **accessible to everyone** — from researchers and engineers to farmers and policy makers — without needing a laboratory or years of expertise.

### Scientific Foundation of Training Data
Every data point in the 500K+ training set was mathematically generated by simulating real anaerobic digestion laws:

- **Buswell Stoichiometry** — calculates theoretical methane potential (BMP) from elemental composition (C, H, O, N, S)
- **C/N Ratio Kinetics** — models microbial nutritional balance; optimal range 20–30 for methanogenesis
- **Monod-type Growth Kinetics** — simulates microbial growth rate response to temperature and substrate concentration
- **pH Inhibition Functions** — captures inhibitory effects on acetogenesis and methanogenesis
- **Mass Balance Equations** — ensures volatile solids reduction is thermodynamically consistent
- **Natural Anaerobic Digestion Process Laws** — governs behavioural predictions when operational parameters change

## 🧠 ML Pipeline Architecture

BioSmart uses a **sequential chained prediction pipeline** — each stage feeds its output as an input feature into the next stage, progressively enriching the feature space. There is no parallel ensemble or inter-model blending — each stage uses a single independently tuned best model.

| Stage | Model | Type | Output | Notes |
|---|---|---|---|---|
| **Stage 1** | XGBoost Classifier | Classification | `run_status` | Predicts SUCCESS / PARTIAL / FAIL |
| **Stage 2** | CatBoost Classifier | Classification | `primary_failure_cause` | Runs only on non-SUCCESS rows (8 failure classes) |
| **Stage 3** | XGBoost Regressor | Regression | `total_biogas_L` | Uses Stage 1 output as `chain_status` input feature |
| **Stage 4** | CatBoost Regressor | Regression | `CH4_pct` | Uses Stage 3 output as `chain_yield` input feature |

### How Chaining Works
- Stage 1 predictions are reshaped and appended to the base features as `chain_status` → fed into Stage 3
- Stage 3 predictions are appended as `chain_yield` → fed into Stage 4
- This allows downstream regression models to condition on upstream classification results, improving accuracy without requiring a separate meta-learner

### Output Variables Predicted
- 🚦 **Run Status** — SUCCESS (✅) / PARTIAL (⚠️) / FAIL (🔴)
- ⚠️ **Primary Failure Cause** — One of 8 classified failure categories (for non-SUCCESS cases only)
- ⚡ **Total Biogas Yield (litres/day)** — total volumetric gas production from the digester
- 🔥 **Methane Content (CH₄ %)** — fuel quality of the produced gas

## 🔧 Hyperparameter Tuning

Each model was independently tuned using **RandomizedSearchCV on a 10% subsample** of the training data, then retrained at full capacity with the best parameters found.

### Tuning Strategy
- **Search method:** RandomizedSearchCV (not GridSearch — computationally efficient for large datasets)
- **Tuning estimators:** 500 (for hyperparameter ranking)
- **Final training estimators:** 5000 (full-capacity retraining with best params)
- **No early stopping during tuning** — eval_set is not passed in RandomizedSearchCV; early stopping is reserved for final training only

### Classifier Tuning Grids

| Parameter | XGBoost Classifier | CatBoost Classifier |
|---|---|---|
| Depth / max_depth | 5, 6, 8 | 5, 6, 8 |
| learning_rate | 0.03, 0.07, 0.15 | 0.03, 0.07, 0.15 |
| subsample | 0.7, 0.85, 1.0 | — |
| colsample_bytree | 0.7, 0.85, 1.0 | — |
| min_child_weight | 5, 10, 20 | — |
| gamma | 0, 0.1, 0.3 | — |
| l2_leaf_reg | — | 1, 3, 5, 10 |
| bagging_temperature | — | 0.0, 0.5, 1.0 |

### Regressor Tuning Grids

### Class Imbalance Handling
The training dataset contains a significant class imbalance in `run_status` (~3.7% SUCCESS rows). To address this, CatBoost Classifier uses `auto_class_weights="Balanced"` — which automatically computes inverse-frequency weights for each class, preventing the model from being biased toward the majority class.

## 📐 Feature Engineering — 37-Column Feature Vector

The `engine.py` module constructs a precisely engineered 37-column input vector for the ML pipeline:

### Raw Operating Parameters (20 features)
`n_feedstock_components`, `is_continuous`, `water_added_kg`, `total_feed_mass_kg`,
`digester_volume_m3`, `HRT_days`, `operating_temp_C`, `temp_fluctuation_degC`,
`operating_pressure_kPa`, `feed_flow_rate_kg_per_day`, `ISR_dimensionless`,
`urea_added_kg`, `blended_CN_ratio`, `TS_pct`, `VS_TS_pct`, `total_VS_kg`,
`blended_lignin_pct`, `OLR_kg_VS_m3_day`, `ammonia_risk_score`, `H2S_risk_score`

### Biochemical Composition (4 features)
`blended_fat_pct`, `blended_protein_pct`, `blended_carb_pct`, `CH4_stoich_index`

### Physics Efficiency Factors (13 features)
`phys_eff_CN`, `phys_eff_OLR`, `phys_eff_temp_base`, `phys_eff_temp_fluctuation`,
`phys_eff_HRT`, `phys_eff_TS`, `phys_eff_ISR`, `phys_eff_pressure`,
`phys_eff_lignin`, `overall_efficiency`, `geom_mean_efficiency`,
`eff_mode`, `overall_efficiency_adjusted`

> The physics efficiency factors are derived directly from anaerobic digestion kinetics and process engineering equations — each captures how far a given parameter deviates from its biological optimum. This is the core Chemical Engineering contribution to the feature space.

## ⚗️ About the Training Data

This model is trained on **physics-informed synthetic data** — not arbitrary random numbers. Every data point was mathematically generated by simulating the natural process of anaerobic digestion:

- Elemental composition (C, H, O, N, S) sourced from published literature values for each feedstock
- Buswell equation applied to derive theoretical biochemical methane potential (BMP)
- Monod kinetics used to model microbial growth rates across temperature and substrate gradients
- pH inhibition sigmoid functions applied to acetogenesis and methanogenesis stages
- C/N ratio constraints enforced to simulate realistic microbial nutritional balance
- Operational parameter ranges bounded by real-world digester engineering limits

While synthetic in origin, the dataset follows the **same thermodynamic and kinetic laws** governing real digesters. This approach is widely used in process engineering to generate robust training data when large-scale experimental datasets are unavailable.

> Predictions are **scientifically grounded estimates** — reliable for feasibility studies, comparative analysis, process planning, and research-level estimation.
